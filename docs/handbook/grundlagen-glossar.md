# Glossar – die wichtigsten Begriffe

Dieses Vokabular gilt im ganzen Handbuch und in beiden Apps. Wenn ein Wort hier steht, ist immer
genau das gemeint – es gibt keine Synonyme.

| Begriff | Bedeutung |
|---|---|
| **Beleg** | Die kleinste Arbeitseinheit: ein Wareneingangs-Vorgang (ein Lieferschein-Abschnitt). Ein Beleg hat einen Lagerplatz, Positionen und einen Status. Das ist das, was eine Person abholt und bearbeitet. Auf dem Bildschirm meist als **`'WE ...'`** (WE-Beleg-Nummer) sichtbar. |
| **Bündel** | Die Tagesmenge an Belegen für **genau eine** Person, in Abhol-Reihenfolge. Pro Person und Tag gibt es **höchstens ein** Bündel. „Einen Beleg zuweisen" heißt: den Beleg in das Bündel dieser Person legen. |
| **Position** | Eine Artikelzeile innerhalb eines Belegs (Artikel, Farbe, Größe, Menge). Ein Beleg hat eine oder mehrere Positionen. |
| **Lagerplatz** | Der Ort, an dem die Ware liegt (z. B. `'R3'`, `'HB-3'`, `'P-2'`). |
| **Bereich** | Die feste Arbeits-/Skill-Klasse eines Belegs. Sie ergibt sich **automatisch aus der Art des Lagerplatzes** und ist **kein** frei pflegbarer Katalog. Es gibt genau drei: **Regal**, **Palette**, **Hängebahn**. |
| **Warenart** | Fachliche Einordnung eines Belegs/einer Position, z. B. `'Vororder'`, `'Nachorder'`, `'NOS'`, `'Extrabestellung'`. Hilft beim Selbst-Priorisieren. |
| **Lieferung / Gruppe** | Mehrere Belege, die zusammen als **eine physische Anlieferung** erkannt wurden (z. B. mehrere Kartons eines Lieferscheins). Sie sollen bei **einer** Person bleiben. Auf dem Bildschirm als **`'Lieferung ×n'`** sichtbar. |
| **Pool** | Die Menge der freien, noch nicht zugeteilten Belege, die verteilt werden können. |
| **Automatik** | Die automatische, faire Verteilung der freien Belege nach Schichtplan und Priorität. Im Cockpit über `'Automatik An'` / `'Automatik Aus'` steuerbar. |
| **Prüfstufe** | Wie genau die Ware im Wareneingang geprüft wird: `'Nein'`, `'10 %'`, `'20 %'`, `'Voll'`. „Nein" heißt **nicht** „gar nichts prüfen" – siehe Kapitel A3. |
| **Teilabschluss** | Nur ein Teil eines Belegs wird abgeschlossen; die Restware geht (meist am Folgetag) zurück in die Planung. |
| **Parken** | Einen Beleg vorübergehend aus der Verteilung nehmen. Als Mitarbeiter: den Rest deines Wagens parken, wenn er voll ist. Als Teamlead: einen Beleg gezielt zurückstellen. |
| **Zurück an Bucher** | Ein Beleg, dem Pflichtdaten fehlen (z. B. Lagerplatz oder Lieferschein-Nr), wird nicht verteilt, sondern muss von der buchenden Stelle korrigiert werden. |
| **ZST / Tagesabschluss** | Der Tagesabschluss: fertige Belege werden ans Altsystem übergeben und damit endgültig abgeschlossen. |
| **Skill-Stufe (Tier)** | Erfahrungsstufe einer Person: `'Profi'`, `'Fortgeschritten'`, `'Basis'`, `'Starter'`, `'Dummy'`. Sie steuert, ob die Automatik der Person automatisch Arbeit geben darf. |
| **Override** | Ein manueller Teamlead-Eingriff mit **Pflicht-Grund** (z. B. „Beleg zuweisen", „Beleg entziehen"). Jeder Override wird protokolliert. |

> **Hinweis zum Wort „Paket":** „Paket" ist **kein** Fachbegriff dieser Anwendung, sondern nur Teil
> des Produktnamens (Paketlagerdispo). Die Tagesmenge einer Person heißt immer **Bündel**, die
> einzelne Arbeitseinheit immer **Beleg**.
